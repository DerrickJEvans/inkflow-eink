🔗 Connection Guide

Configure your ESP32 Waveshare display by pointing the C++ client to:

http://[server-ip]:5000/api/display/raw?device=esp32_screen&width=400&height=300

ESP32 downloads pre-dithered raw bytes directly over WiFi into screen SRAM for quick, low-power renders.

Configure the Python client on a Pi Zero by pointing the connection string in config to:

http://[server-ip]:5000/api/display/image.png?device=pi_screen&width=800&height=480

Pulls full dithered PNGs via standard HTTP GET. Supports Waveshare, Pimoroni Inky, and local mock previews.

To use an official TRMNL screen, access your device's WiFi Pairing Portal and configure:
